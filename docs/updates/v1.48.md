# March 2026

Install with: [GitHub Actions](../install.md#with-github-actions) | [Azure Pipelines](../install.md#with-azure-pipelines) | [PowerShell](../install.md#with-powershell)

---

Welcome to the March 2026 release of PSRule for Azure.
This release includes new features, new rules and improvements to existing rules.

See the [change log][1] or [GitHub history][2] for a complete list of changes.

## Experimental support for Terraform Plans

This release includes experimental support for expanding and consuming Terraform plan JSON files for resources using the AzAPI provider.
Previously using PSRule for Azure to analyze Terraform infrastructure as code (IaC) was not supported.
Many in the community have expressed interest in adding support for Terraform.

You can now use PSRule for Azure to analyze Terraform plans prior to deployment _(pre-flight)_.

You can read more about this feature and how to get started in the [Using Terraform][3] documentation.

  [3]: using-terraform.md

## New baselines

This release adds March 2026 built-in baselines: `Azure.GA_2026_03`, `Azure.Preview_2026_03`, and `Azure.CAF_2026_03`.
These baselines include rules released through March 2026.
The previous `Azure.GA_2025_12` and `Azure.Preview_2025_12` baselines have been marked obsolete.

## Security hardening

New rules to help reduce supply-chain and runtime risks include:

- `Azure.Automation.RunbookPinned` &mdash; ensures runbook external scripts use pinned commit hashes.
- `Azure.DeploymentScript.Pinned` &mdash; ensures deployment script external URIs use pinned commit hashes.
- `Azure.Fleet.PublicKey` &mdash; checks Linux Azure Fleet VM profiles use SSH public-key authentication (password auth disabled).
- `Azure.VM.SecureBoot` / `Azure.VMSS.SecureBoot` &mdash; ensure Secure Boot is enabled for Virtual Machines and Virtual Machine Scale Sets.

Additionally, new rules were added for:

- To ensure Container Registry audit diagnostic logs are enabled.
- To ensure Container App liveness and readiness probes use HTTP checks for HTTP-based ingress.
- To ensure Service Bus geo-replication replica locations are within allowed regions.
